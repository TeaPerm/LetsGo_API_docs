# Pagination

In this guide, we will look at how to work with paginated responses when querying the LetsGo API. By default, there is no response limit. However, the user can add limit by adding `limit` parameter to your requests. {{ className: 'lead' }}

When an API response returns a list of objects, on some endpoints pagination is supported. 

## Example using cursors

<Row>
  <Col>

    In this example, we request the page `2` limit `4` from the `/products` endpoint. As a result, we get the second set of four prodcuts.

    <Properties>
      <Property name="page" type="int">
         The page of the products returned.
      </Property>
      <Property name="limit" type="int">
        The limit the number of products returned.
      </Property>
    </Properties>

  </Col>

    ```bash {{ title: 'Manual pagination using cURL' }}
    curl -G https://lets-go-phi.vercel.app/products \
      -d page=2 \
      -d limit=4
    ```
